# ZebraPuzzleTask
## Теория
При решении многих прикладных задач, связанных с применением булевых функций, возникает необходимость в оптимизации механизмов работы с имеющимися программными и 
аппаратными ресурсами. Следовательно данные нужно хранить в компактном виде и обрабатывать их за минимальное время с требуемой точностью. Решением этой проблемы могут быть бинарные решающие диаграммы 
(BDD – Binary Decision Diagrams), являющиеся одной из самых распространенных форм представления логический функций и удовлетворяющих описанным выше требованиям к эффективности. 
Я исследовал BDD на примере решения частного случая класса задач программирования в ограничениях - известной логической головоломки «Задача Эйнштейна» ([подробнее о задаче)](https://thecode.media/einstein/)).

## Решаемое задание
Пусть имеется N объектов, расположенных определенным образом. Далее пусть имеется M свойств, принимающих N различных значений. На основе некоторого набора данных о системе необходимо восстановить ее полное исходное состояние. Для этого применяются различные типы ограничений.

*В результате нужно найти все возможные решения задачи с исходными данными.*
